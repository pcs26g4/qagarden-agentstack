2. LOCATOR USAGE (CRITICAL):

3. JSON TESTCASES USAGE (CRITICAL):
   - JSON testcases contain specific inputs for each test
   - USE the actual input values from each test case
   - DO NOT use hardcoded values - use the provided inputs
   - Each test function should use its specific test case inputs
   - Convert each JSON test case into a separate test function with its inputs
   - NO runtime JSON loading

4. TEST STRUCTURE & FIXTURE USAGE (CRITICAL):
   - One test function per JSON test case
   - Function names based on test case ID: test_page_valid_credentials, test_page_empty_email, etc.

5. IMPORTS:

7. OUTPUT FORMAT:
