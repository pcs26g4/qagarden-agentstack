TASK:
Generate clean, executable Playwright pytest tests by converting JSON test cases into individual test functions.

CRITICAL RULES (MANDATORY):

1. FIXTURE USAGE:
   - NEVER define @pytest.fixture in test files
   - conftest.py already provides browser, context, page fixtures
   - Use only: def test_function_name(page):
   - NEVER use: with sync_playwright() as p:

2. LOCATOR USAGE (MANDATORY & AOM-FIRST):
   - The LOCATORS JSON contains key-value pairs where the VALUE is a full Playwright expression.
   - Example JSON: "LOGIN_BUTTON": "page.getByRole('button', { name: 'Login' })"
   - To use this, simply use the expression directly. Example: `page.get_by_role('button', name='Login').click()`
   - NEVER wrap these in `page.locator()` like `page.locator("page.getByRole(...)")` - that is invalid.
   - If the value in JSON is a CSS selector (e.g. "#id"), then use `page.locator("#id")`.
   - Priority Matrix: ALWAYS prefer AOM Locators (`get_by_role`, `get_by_label`, `get_by_text`) over raw CSS or XPath when available. This makes the tests unbreakable.
   - Trust the LOCATORS JSON values as the primary source for identifying elements.

3. JSON TESTCASES USAGE (CRITICAL):
   - JSON testcases contain specific inputs for each test
   - USE the actual input values from each test case
   - DO NOT use hardcoded values - use the provided inputs
   - Each test function should use its specific test case inputs
   - Convert each JSON test case into a separate test function with its inputs
   - NO runtime JSON loading

4. TEST STRUCTURE, FIXTURES, & MARKERS (CRITICAL):
   - One test function per JSON test case.
   - Function names based on test case ID: `test_page_valid_credentials`, `test_page_empty_email`.
   - MARKERS: You MUST semantically tag every test using `@pytest.mark`.
     * If it's a critical path (e.g., Login, Navigation), tag it `@pytest.mark.smoke`.
     * If it's an edge case, search, or pagination, tag it `@pytest.mark.regression`.
   - FIXTURE RULES:
     * Simple pages: `def test_function(page):` + `page.goto(URL_VARIABLE)`
     * Auth pages: `def test_function(authenticated_page):` + `authenticated_page.goto(URL_VARIABLE)`

5. IMPORTS:

6. PYTHON SYNTAX RULES (CRITICAL):
   - NEVER use JavaScript regex syntax like /pattern/
   - ALWAYS use Python regex: re.compile(r"pattern") or re.search(r"pattern", string)
   - For URL pattern matching: re.compile(r"accounts\.google\.com")
   - ALWAYS import 're' module when using regex patterns
   - NO JavaScript syntax allowed - this is Python only
   - Use proper Python string escaping with raw strings r"pattern"
   - Example CORRECT: expect(page).to_have_url(re.compile(r"accounts\.google\.com"))
   - Example WRONG: expect(page).to_have_url(/accounts\.google\.com/)
   - PLAYWRIGHT METHODS (CRITICAL):
     * ALWAYS use snake_case: page.get_by_role(...), page.get_by_text(...)

     * Attribute errors like 'Page object has no attribute getByRole' mean you used camelCase. STOP IT.
   - ASSERTION RULES (CRITICAL):
     * ALWAYS use `expect(page).to_have_url(URL_VARIABLE)` to verify navigation.
     * NEVER use `expect(page.url).to_equal(...)` -> THIS WILL FAIL with ValueError.
     * NEVER use `assert page.url == ...` -> This is flaky. Use `expect(page).to_have_url(...)`.
     * For button visibility: `expect(page.locator("#id")).to_be_visible()`.
